# 📚 자바 제7장: 상속 (Inheritance) - 심화 및 핵심 요약

상속은 부모 클래스의 멤버를 자식에게 물려주어 코드의 중복을 제거하고 다형성을 구현하는 토대가 됩니다.

### 1. 상속의 기본 규칙
* **extends 키워드:** 클래스 이름 뒤에 사용하여 부모 클래스를 지정합니다.
* **단일 상속:** 자바는 한 클래스가 여러 부모를 상속받는 다중 상속을 지원하지 않습니다.
* **상속 제외:** 부모의 `private` 멤버와 생성자는 상속 대상에서 제외됩니다.

### 2. 생성자 호출과 super()
* **실행 순서:** 자식 객체를 생성하면 부모 객체가 먼저 생성된 후 자식 객체가 생성됩니다.
* **super():** 자식 생성자의 첫 줄에서 부모 생성자를 호출합니다. 매개변수가 있는 부모 생성자만 존재할 경우, 자식 생성자에서 `super(인자, ...)`를 반드시 명시해야 합니다.

### 3. 메소드 오버라이딩 (Method Overriding)
* **정의:** 상속받은 부모 메소드를 자식 클래스에서 재정의하는 것입니다.
* **표시:** `@Override` 어노테이션을 사용하여 컴파일러가 정확히 재정의되었는지 체크하게 합니다.
* **규칙:** 부모 메소드와 이름, 리턴 타입, 매개변수가 같아야 하며, 접근 제한을 더 강하게(좁게) 변경할 수 없습니다.

### 4. 타입 변환 (Type Casting)
* **자동 타입 변환 (Promotion):** 자식 객체는 부모 타입의 변수에 대입될 수 있습니다. 이때 부모 클래스에 선언된 멤버만 접근 가능하지만, 오버라이딩된 메소드는 자식의 것이 실행됩니다.
* **강제 타입 변환 (Casting):** 부모 타입을 다시 자식 타입으로 바꿀 때 사용하며, 원래 자식 객체였던 경우에만 가능합니다.
* **instanceof:** 변수가 가리키는 실제 객체의 타입을 확인할 때 사용하여 런타임 에러를 방지합니다.

---

### 💻 7장 핵심 코드 예시 (타입 변환 & 다형성)

```java
// 부모 클래스
class Vehicle {
    void run() {
        System.out.println("차량이 달립니다.");
    }
}

// 자식 클래스 1
class Bus extends Vehicle {
    @Override
    void run() {
        System.out.println("버스가 도로를 달립니다.");
    }
    
    void checkFare() {
        System.out.println("승차 요금을 체크합니다.");
    }
}

// 자식 클래스 2
class Taxi extends Vehicle {
    @Override
    void run() {
        System.out.println("택시가 빠르게 달립니다.");
    }
}

public class Main {
    public static void main(String[] args) {
        // 1. 자동 타입 변환 및 오버라이딩 확인
        Vehicle v1 = new Bus();
        v1.run(); // 자식인 Bus의 run()이 실행됨 (다형성)
        // v1.checkFare(); // 에러: Vehicle 타입이라 자식 전용 메소드 사용 불가
        
        // 2. instanceof 및 강제 타입 변환
        if (v1 instanceof Bus) {
            Bus bus = (Bus) v1;
            bus.checkFare(); // 이제 자식 전용 메소드 사용 가능
        }
        
        // 3. 매개변수의 다형성
        drive(new Taxi());
        drive(new Bus());
    }
    
    // 부모 타입인 Vehicle을 매개변수로 받아 모든 자식 객체를 수용
    static void drive(Vehicle v) {
        v.run();
    }
}
